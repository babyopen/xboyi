# 生肖预测模块完整逻辑分析

## 📊 数据流程图

```
历史开奖数据
    ↓
calcContinuousScores() - 连续线性评分算法
    ↓
五维度评分系统
    ├─ 1. 基础热度分 (30分)
    ├─ 2. 形态共振分 (20分)
    ├─ 3. 间隔规律分 (20分)
    ├─ 4. 趋势动量分 (15分)
    └─ 5. 近期动量分 (15分)
    ↓

    ↓

    ↓

```

## 🎯 核心算法详解

### 一、基础热度分（30分）

**计算依据**：生肖遗漏期数（多少期未出现）

**评分规则**：
```javascript
遗漏期数 ≤ 2期  → 25-30分（极热状态）
遗漏期数 3-6期  → 18-25分（温热状态）
遗漏期数 7-12期 → 10-18分（温和状态）
遗漏期数 13-20期→  4-10分（偏冷状态）
遗漏期数 > 20期 →  2-4分 （极冷反弹潜力）
```

**示例**：
- 狗：遗漏0期（上期刚出）→ 30分
- 鼠：遗漏2期 → 25分
- 牛：遗漏3期 → 23.25分 ≈ 23分

**标签显示**：`热28`、`温20`、`冷13`

---

### 二、形态共振分（20分）

**计算依据**：多维度属性匹配度

#### 2.1 单双大小共振（6分）
- 分析最近15期的单双、大小比例
- 确定热门形态（如"单"占60%、"大"占55%）
- 生肖号码属性与热门形态匹配则加分

**示例**：
- 如果最近15期"单"占优，狗的号码多为双数 → 不匹配
- 鼠的号码多为单数 → 匹配得高分

#### 2.2 波色匹配（4分）
- 分析最近20期的红蓝绿波色分布
- 确定热门波色
- 生肖主色调与热门波色匹配则加分

#### 2.3 上期尾数关联（3分）
- 根据上期特码尾数，查找关联生肖
- 映射关系：
  ```
  尾数0 → 鼠、猪
  尾数1 → 牛、狗
  尾数2 → 虎、鸡
  尾数3 → 兔、猴
  尾数4 → 龙、羊
  尾数5 → 蛇、马
  ...
  ```

#### 2.4 连出/遗漏形态（3分）
- 最近5期内出现≥2次 → +3分（连出）
- 遗漏≥15期 → +2分（高遗漏反弹）

#### 2.5 五行关系（4分）
- 相生关系（金生水、水生木...）→ +2分
- 相同五行 → +1分
- 当前五行是热门五行 → +2分

**标签显示**：`形3`、`形8`、`形9`

---

### 三、间隔规律分（20分）

**计算依据**：生肖之间的位置间隔模式

**算法步骤**：
1. 统计最近50期的生肖转换间隔
2. 找出最常见的5个间隔模式（如+2、-3、+5等）
3. 根据上期生肖，计算各生肖的间隔值
4. 匹配常见间隔则给高分

**示例**：
- 上期是"狗"（索引10）
- 常见间隔是+2（狗→鼠）
- 鼠的间隔值 = (0 - 10 + 12) % 12 = 2 → 匹配 → 高分
- 牛的间隔值 = (1 - 10 + 12) % 12 = 3 → 不匹配 → 低分

**标签显示**：`间20`、`间15`、`间0`

---

### 四、趋势动量分（15分）

**计算依据**：短期 vs 中期的出现频率对比

**算法**：
```javascript
recentCount = 最近10期出现次数
prevCount = 第11-20期出现次数

if (recentCount > prevCount) {
  trendScore = min(8, (recentCount - prevCount) * 4)  // 上升趋势
} else if (recentCount < prevCount) {
  trendScore = max(-4, (recentCount - prevCount) * 2)  // 下降趋势（可能为负）
} else {
  trendScore = 0  // 平稳
}
```

**标签显示**：`趋4`、`趋8`、`趋0`

---

### 五、近期动量分（15分）

**计算依据**：最近3-7期是否出现过

**评分规则**：
```javascript
最近3期出现过     → 7分（强动量）
7期内出现过       → 4分（中等动量）
超过7期未出现     → 2分（弱动量）
```

**标签显示**：`动7`、`动4`、`动2`

---

## 📈 总分计算

```
总分 = 基础分(30) + 形态分(20) + 间隔分(20) + 趋势分(15) + 动量分(15)
     = 100分（理论最大值）
```

## 🔄 数据同步机制

### 自动保存流程
```javascript
// 1. 计算连续评分
const continuous = analysisCalc.calcContinuousScores(data);

// 2. 按总分排序
const sortedZodiacs = Object.entries(continuous.scores)
  .sort((a, b) => b[1] - a[1]);

// 3. 更新data对象
const updatedData = {
  ...data,
  sortedZodiacs: sortedZodiacs
};

// 4. 触发自动保存
analysisRender.autoSaveZodiacPrediction(updatedData);
```

### 保存到记录页面
- 使用 `IssueManager.getNextIssue()` 获取预测期号
- 保存前6名生肖及其分数
- 触发自定义事件 `zodiacPredictionSaved`
- 跨标签页同步（Storage事件）

## ✅ 逻辑检查结果

### 优点
1. ✅ **多维度评分**：综合考虑热度、形态、间隔、趋势、动量
2. ✅ **非线性映射**：基础分使用S型曲线，更符合实际规律
3. ✅ **动态权重**：根据数据分布自动调整评分
4. ✅ **详细标签**：用户可以清楚看到每个维度的得分
5. ✅ **自动保存**：预测结果自动同步到记录页面

### 潜在优化点
1. ⚠️ **趋势分可能为负**：虽然最终用`Math.max(0, trendScore)`限制，但逻辑上可以更清晰
2. ⚠️ **间隔分计算复杂**：涉及模运算和环形距离，可能需要更多注释
3. ⚠️ **形态分上限检查**：虽然有`Math.min(score, 20)`，但各维度累加可能超过20

### 建议改进
```javascript
// 当前代码（第1119行）
trendScores[z] = Math.max(0, trendScore);

// 建议改为更清晰的表达
trendScores[z] = Math.max(0, Math.min(15, trendScore)); // 明确上限15分
```
