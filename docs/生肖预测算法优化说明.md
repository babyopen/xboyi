# 生肖预测算法优化说明

## 📊 算法升级概览

本次优化将原有的**3维度评分模型**升级为**5维度综合评分模型**，充分利用分析页面和维度分析页面的所有数据维度。

---

## 🎯 核心改进

### 1️⃣ **基础热度分（30分）** - 非线性S型曲线映射

**原算法问题：**
- 简单的线性排名映射（第1名35分 → 第12名0分）
- 无法区分"极热"和"温热"的差异
- 忽略了长期遗漏生肖的反弹潜力

**新算法优势：**
```javascript
// S型曲线分段映射
- 极热状态（0-2期遗漏）：25-30分
- 温热状态（3-6期遗漏）：18-25分  
- 温和状态（7-12期遗漏）：10-18分
- 偏冷状态（13-20期遗漏）：4-10分
- 极冷反弹（>20期遗漏）：2-4分
```

**科学依据：**
- 近期高频出现的生肖保持高热度的概率更高
- 长期遗漏的生肖有反弹可能，但不应给予过高分数
- 避免了线性映射中"中等遗漏"得分过低的问题

---

### 2️⃣ **形态共振分（20分）** - 多维度形态匹配

**新增维度：**

#### ① 单双大小共振（6分）
- 分析最近15期的单双、大小比例
- 根据热门形态的占比动态加权
- 示例：如果"单大"占比65%，匹配的生肖获得更高分数

#### ② 波色趋势匹配（4分）
- 统计最近20期的红/蓝/绿波色分布
- 生肖对应号码的主色调与热门波色匹配时加分
- 根据波色占比动态调整分数（2-4分）

#### ③ 上期尾数关联（3分）
- 保留原有逻辑：基于尾数→生肖映射表
- 例如：上期尾数5 → 蛇/马获得3分

#### ④ 连出/遗漏形态（3分）
- 连出加分：最近5期出现≥2次 → +3分
- 遗漏反弹：遗漏≥15期 → +2分
- 区分"持续热"和"冷号反弹"两种模式

#### ⑤ 五行关系网络（4分）
- **相生关系**（+2分）：金→水→木→火→土→金
- **相同五行**（+1分）：上期与当前生肖五行相同
- **热门五行加成**（+2分）：当前生肖五行是近期热门五行

**原算法对比：**
- 原算法仅15分上限，且维度单一
- 新算法增加波色、五行热门度等维度
- 动态权重分配，更贴合实际数据趋势

---

### 3️⃣ **间隔规律分（20分）** - 增强版模式识别

**改进点：**

#### ① 扩大历史窗口
- 从30期扩大到50期
- 捕捉更长周期的间隔规律

#### ② 增加容错范围
- 从TOP 3常见间隔增加到TOP 5
- 提高匹配成功率

#### ③ 频率加权评分
```javascript
// 原算法：匹配=20分，不匹配=0分（二元判断）
// 新算法：根据出现频率动态评分
const frequency = matched.count / maxIntervalCount;
score = 10 + frequency * 10; // 10-20分区间
```

**实际效果：**
- 高频间隔模式（如+2位置）获得18-20分
- 中频间隔模式获得14-17分
- 低频但仍在前5的获得10-13分

---

### 4️⃣ **趋势动量分（15分）** - 🆕 全新维度

**设计理念：**
借鉴金融技术分析中的"动量指标"，捕捉生肖的短期趋势变化。

#### ① 短期趋势（0-8分）
```javascript
// 对比最近10期 vs 之前10期的出现次数
const recentCount = 最近10期出现次数;
const prevCount = 前10-20期出现次数;

if (recentCount > prevCount) {
  trendScore = (recentCount - prevCount) * 4; // 上升趋势
} else {
  trendScore = (recentCount - prevCount) * 2; // 下降趋势（负值）
}
```

**意义：**
- 捕捉"升温"或"降温"的趋势
- 上升趋势最高+8分，下降趋势最低-4分

#### ② 动量指标（2-7分）
```javascript
if (最近3期出现过) {
  momentumScore = 7; // 强动量
} else if (7期内出现过) {
  momentumScore = 4; // 中等动量
} else {
  momentumScore = 2; // 弱动量
}
```

**意义：**
- 衡量生肖的"活跃度"
- 近期活跃的生肖更可能继续出现

---

### 5️⃣ **总分计算**

```
总分 = 基础分(30) + 形态分(20) + 间隔分(20) + 趋势分(15) + 动量分(15)
     = 100分满分
```

**与原算法对比：**
- 原算法：35 + 15 + 20 = 70分满分
- 新算法：30 + 20 + 20 + 15 + 15 = 100分满分
- 分数分布更均匀，区分度更高

## 🔬 算法优势总结

### ✅ 科学性提升
1. **多维度融合**：整合了分析页面的所有统计数据
2. **动态权重**：根据数据分布自动调整各维度权重
3. **非线性映射**：S型曲线更符合实际概率分布

### ✅ 可解释性增强
1. **透明评分**：每个维度的分数清晰可见
2. **标签化展示**：用户可直观理解评分来源
3. **状态标识**：热/温/冷三级分类

### ✅ 适应性更强
1. **趋势捕捉**：新增趋势和动量维度
2. **容错机制**：间隔匹配从TOP3扩展到TOP5
3. **平衡冷热**：既关注热号持续性，也考虑冷号反弹

## ⚙️ 技术实现细节

### 缓存机制
- 保留了原有的缓存逻辑
- 以`analyzeLimit`为缓存key
- 检测到历史数据更新时自动失效

### 性能优化
- 使用`PerformanceMonitor`监控计算耗时
- 避免重复计算：一次性遍历获取多个统计值
- 数组操作优化：减少不必要的排序和过滤

### 兼容性
- 完全兼容现有UI结构
- 向后兼容：即使缺少某些数据也不会报错
- 渐进式增强：新维度分数为0时不影响整体评分
