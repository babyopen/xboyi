# 精选特码历史分别保存功能说明

## 功能概述

精选特码历史记录现在可以根据不同的筛选条件（期数 × 号码数量）分别保存，而不是互相覆盖。

## 支持的筛选条件组合

### 期数选项（4种）
- 10期
- 20期
- 30期
- 全年

### 号码数量选项（4种）
- 5个
- 10个
- 15个
- 20个

### 总计
同一期号最多可以保存 **16条** 不同的记录（4种期数 × 4种号码数量）

## 数据结构

每条记录包含以下字段：

```javascript
{
  issue: "2024001",           // 期号
  period: 10,                 // 期数范围：10/20/30/all
  numCount: 5,                // 号码数量：5/10/15/20
  numbers: ["01", "13", ...], // 预测号码
  hotNumbers: [...],          // 热号数据
  coldNumbers: [...],         // 冷号数据
  mode: "hot",                // 模式：hot/cold/auto
  type: "hot",                // 类型
  checked: false,             // 是否已核对
  matched: null,              // 是否命中
  actualNumbers: [],          // 实际开奖号码
  createdAt: 1234567890,      // 创建时间
  updatedAt: 1234567890       // 更新时间
}
```

## 核心修改

### 1. 保存逻辑（storage.js - saveNumberRecord）

**去重规则：**
```javascript
// 使用复合键：期号 + 期数范围 + 号码数量
const existingIndex = records.findIndex(r => 
  r.issue === recordData.issue && 
  r.period === recordData.period && 
  r.numCount === recordData.numCount
);
```

**效果：**
- 同一期号的不同筛选条件会保存为独立记录
- 相同筛选条件的记录会被更新（不会重复保存）

### 2. 自动保存逻辑（analysis-render.js - autoSaveSpecialRecord）

**新增内容：**

// 保存到记录中

### 3. 渲染逻辑（record.js - renderSpecialHistory）

**元数据显示：**
```html
<div class="history-meta">
  <span>📊 10期</span>
  <span>🎯 5个</span>

```

用户可以清楚看到每条记录对应的筛选条件。

### 4. 删除逻辑（record.js - _deleteSpecialRecord）

**删除规则：**
```javascript
// 使用复合键删除
if (record.period && record.numCount) {
  return !(r.issue === record.issue && 
           r.period === record.period && 
           r.numCount === record.numCount);
}
// 兼容旧数据
return r.issue !== record.issue;
```

### 5. 核对逻辑（storage.js - checkNumberRecord）

**核对规则：**
- 核对时会更新所有匹配该期号的记录
- 可能同时更新多条记录（不同筛选条件）
- 返回更新的记录数量

## 兼容性说明

### 旧数据处理

对于之前保存的旧记录（没有 `period` 和 `numCount` 字段）：

- **保存时**：会自动添加这两个字段
- **删除时**：如果记录没有这两个字段，则只按期号匹配（保持原有行为）
- **核对时**：正常工作，不受影响

### 数据迁移

不需要手动迁移数据。新保存的记录会自动包含 `period` 和 `numCount` 字段，旧记录会在下次保存时被更新。

## 注意事项

1. **存储限制**：仍然只保留最近50条记录
2. **期号格式**：必须使用下一期期号（由 IssueManager.getNextIssue() 提供）
3. **筛选条件来源**：从 StateManager 中读取当前的 `analyzeLimit` 和 `selectedNumCount`
4. **日志输出**：保存、更新、删除操作都会输出日志，方便调试

## 常见问题

### Q: 为什么同一期号会有多条记录？
A: 因为不同的筛选条件（期数和号码数量）会产生不同的预测结果，需要分别保存以便对比。

### Q: 如何区分不同筛选条件的记录？
A: 每条记录下方会显示元数据，如"📊 10期 🎯 5个"，清楚标识筛选条件。

### Q: 旧记录会受影响吗？
A: 不会。旧记录仍然可以正常显示、删除和核对。新保存的记录会自动添加筛选条件信息。

### Q: 如果我想查看所有筛选条件的记录怎么办？
A: 记录页面默认显示所有记录，按时间倒序排列。您可以通过元数据识别不同的筛选条件。
