## 🚀 性能优化技巧

### 1. 批量操作
```javascript
// ✅ 批量清除缓存
const keys = ['key1', 'key2', 'key3'];
keys.forEach(key => Storage.clearCache(key));

// ❌ 多次单独调用
Storage.clearCache('key1');
Storage.clearCache('key2');
Storage.clearCache('key3');
```

### 2. 事件处理器缓存
```javascript
// ✅ 缓存引用
const handler = (e) => doSomething(e);
item.addEventListener('touchstart', handler, { passive: true });
// 后续可以移除
item.removeEventListener('touchstart', handler);

// ❌ 匿名函数(无法移除)
item.addEventListener('touchstart', (e) => doSomething(e), { passive: true });
```

### 3. DOM操作优化
```javascript
// ✅ 使用 DocumentFragment
const fragment = document.createDocumentFragment();
items.forEach(item => {
  const el = createItem(item);
  fragment.appendChild(el);
});
container.appendChild(fragment); // 单次重排

// ❌ 多次DOM操作
items.forEach(item => {
  const el = createItem(item);
  container.appendChild(el); // 多次重排
});
```

## 📱 移动端兼容性

### iOS Safari 特殊处理
```css
/* 消除点击高亮 */
-webkit-tap-highlight-color: transparent;

/* GPU加速 */
will-change: transform;

/* 平滑滚动 */
-webkit-overflow-scrolling: touch;
```

```javascript
// 触摸事件必须使用 passive 选项
element.addEventListener('touchstart', handler, { passive: true });
element.addEventListener('touchmove', handler, { passive: false }); // 需要preventDefault时用false
```

### Android Chrome 优化
```css
/* 禁用文本选择 */
user-select: none;
-webkit-user-select: none;

/* 硬件加速 */
transform: translateZ(0);
```

## ⚡ 常见问题解决

### Q1: 滑动删除不流畅?
**A**: 检查以下几点:
1. CSS中是否添加了 `will-change`
2. 是否有过多的DOM节点
3. 是否在touchmove中执行了复杂计算

### Q2: 内存占用持续增长?
**A**: 确保在重新渲染前清理事件绑定:
```javascript
oldItems.forEach(item => record._unbindSwipeDelete(item));
```

### Q3: iOS上点击有延迟?
**A**: 添加CSS:
```css
.clickable {
  cursor: pointer;
  -webkit-tap-highlight-color: transparent;
}
```

### Q4: 数据更新后UI没变化?
**A**: 确保调用了StateManager:
```javascript
Storage.set('key', newData);
StateManager.setState({ key: newData }, false);
renderFunction(); // 重新渲染
```
