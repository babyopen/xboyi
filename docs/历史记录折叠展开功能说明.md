## 📋 功能概述

为记录页面的五个历史记录模块添加了折叠/展开功能，优化页面空间利用，提升用户体验。

**状态说明**：

- `true`: 展开状态（显示全部）

### 2. 核心函数

#### `_applyCollapseLogic(containerId, toggleId, stateKey)`

应用折叠/展开逻辑到指定容器。

**功能**：
1. 获取容器中的所有 `.history-item` 元素
2. 如果记录数 ≤ 2条，隐藏切换按钮
3. 如果记录数 > 2条，显示切换按钮并根据状态控制显示
4. 更新按钮文本（"展开更多" / "收起"）

**代码**：
```javascript
_applyCollapseLogic: (containerId, toggleId, stateKey) => {
  const container = document.getElementById(containerId);
  const toggle = document.getElementById(toggleId);
  
  if (!container || !toggle) return;
  
  const items = container.querySelectorAll('.history-item');
  const isExpanded = record._collapseState[stateKey];
  
  if (items.length <= 2) {
    // 如果记录数不超过2条，隐藏切换按钮
    toggle.style.display = 'none';
    return;
  }
  
  // 显示切换按钮
  toggle.style.display = 'block';
  toggle.textContent = isExpanded ? '收起' : '展开更多';
  
  // 根据状态显示/隐藏多余的记录
  items.forEach((item, index) => {
    if (index >= 2) {
      item.style.display = isExpanded ? 'block' : 'none';
    }
  });
}
```

---

#### `_toggleCollapse(stateKey, containerId, toggleId)`

切换折叠/展开状态。

**功能**：
1. 反转状态值
2. 重新应用折叠逻辑

**代码**：
```javascript
_toggleCollapse: (stateKey, containerId, toggleId) => {
  record._collapseState[stateKey] = !record._collapseState[stateKey];
  record._applyCollapseLogic(containerId, toggleId, stateKey);
}
```

## 🎯 功能特性

### ✅ 默认行为
- **初始加载**：每个列表默认仅显示最新的 **2条** 记录
- **自动判断**：如果记录总数 ≤ 2条，自动隐藏切换按钮
- **按钮位置**：位于列表底部，宽度100%

### ✅ 交互行为
- **点击"展开更多"**：显示所有历史记录，按钮文本变为"收起"
- **点击"收起"**：恢复为仅显示2条记录，按钮文本变为"展开更多"
- **状态保持**：刷新页面后重置为折叠状态

### ✅ UI一致性
- 使用项目现有的 `.toggle-btn` 样式类
- 按钮样式与其他模块保持一致
- 不影响现有CSS样式

## 💡 设计决策

### 为什么只修改3个模块？

**生肖预测历史**和**ML预测历史**已有"展开更多"分页功能：
- 使用分页加载（每次加载5条）
- 通过 `toggleZodiacPredictionHistory()` 和 `toggleMLPredictionHistory()` 加载更多
- 与折叠功能的设计思路不同，因此保留原有实现

### 为什么默认显示2条？

1. **空间优化**：减少初始页面高度，提升首屏体验
2. **信息密度**：2条足够展示最近的预测结果
3. **用户习惯**：符合常见的"查看更多"交互模式
4. **性能考虑**：减少DOM渲染压力

### 为什么不使用CSS动画？

1. **简单高效**：直接使用 `display: none/block` 切换
2. **兼容性**：无需考虑浏览器兼容性
3. **性能**：避免重排重绘开销
4. **需求明确**：用户需要快速展开/收起，不需要过渡效果

---

## ⚠️ 注意事项

1. **状态重置**：刷新页面后，所有模块恢复为折叠状态
2. **动态数据**：当数据更新时（如生成新记录），会自动重新应用折叠逻辑
3. **空数据处理**：如果记录数为0，显示"暂无XX历史"提示，不显示按钮
4. **样式隔离**：折叠逻辑仅控制 `display` 属性，不影响其他CSS样式
5. **事件委托**：使用 `data-action` 属性进行事件委托，避免重复绑定
